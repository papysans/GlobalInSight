# Kiro Steering 文件说明

> 📅 创建日期: 2026-01-23  
> 🎯 目标: 说明 Kiro AI 的 Steering 规则文件

---

## 📚 Steering 文件概览

Steering 文件是 Kiro AI 的行为指南，每次执行时都会自动加载这些规则。

### 当前文件列表

| 文件名 | 用途 | 优先级 |
|--------|------|--------|
| `project-guidelines.md` | 项目开发规范和文档管理 | 最高 |
| `agent-behavior-guide.md` | Agent 行为优化指南 | 高 |
| `mcp-tools-guide.md` | MCP 工具完整使用指南 | 高 |
| `testing-guidelines.md` | 测试规范和最佳实践 | 中 |
| `design-system.md` | 前端设计系统规则 | 中 |

---

## 📖 文件详细说明

### 1. project-guidelines.md
**核心项目规范**

包含内容：
- 📁 核心文档位置
- 📝 文档编写规范
- 🏗️ 技术架构要点
- ⚠️ 重要原则
- 📞 问题排查流程

**何时参考**：
- 开始新任务前
- 创建文档时
- 遇到问题时
- 完成任务后

### 2. agent-behavior-guide.md
**Agent 行为优化**

包含内容：
- 🎯 核心行为原则（主动性、简洁性、效率性、智能性）
- 🚀 工作流程优化
- 🎓 场景化行为指南
- 🔧 工具使用优化
- 📝 沟通风格指南

**何时参考**：
- 每次执行任务时
- 需要优化工作流程时
- 与用户沟通时

### 3. mcp-tools-guide.md
**MCP 工具使用指南**

包含内容：
- 🔧 已配置的 MCP 工具（GitHub、Context7、Firecrawl）
- 📋 工具使用场景矩阵
- 🎯 详细使用指南
- 🚀 自动触发规则
- 🎯 项目特定使用场景

**何时参考**：
- 用户询问技术问题时
- 需要搜索代码时
- 需要最新文档时
- 需要抓取网页时

### 4. testing-guidelines.md
**测试规范**

包含内容：
- 📁 测试目录结构
- 🎯 测试分类和命名规范
- 📝 测试文件模板
- 🚀 运行测试方法
- 🎓 测试编写最佳实践

**何时参考**：
- 编写测试时
- 运行测试时
- 调试测试时

### 5. design-system.md
**前端设计系统**

包含内容：
- 🎨 Token 定义（颜色、间距、字体、圆角）
- 🧩 组件库架构
- 📐 页面结构规范
- 💅 样式架构
- 🎯 业务特定组件

**何时参考**：
- 开发前端组件时
- 设计 UI 时
- 编写样式时

## 🔄 文件更新规则

### 何时更新
- 发现新的最佳实践
- 遇到重复问题
- 项目规范变更
- 新增 MCP 工具

### 更新原则
- 保持文件简洁
- 使用清晰的示例
- 标注更新日期
- 保持一致的格式

### 更新流程
1. 识别需要更新的内容
2. 修改相应的 steering 文件
3. 更新文件底部的"最后更新"日期
4. 如有重大变更，更新本 README
